# Docker (docker)

Плагин Usee: статус демона Docker и контейнеры (CPU/RAM) на дисплее.

## Как это работает
Docker Engine API — это HTTP/1.1. На Windows/Docker Desktop он торчит именованным
пайпом `\\.\pipe\docker_engine`, до которого дотягивается `ctx.pipe` (SDK 3).
`ctx.net.fetch` тут не подходит (только TCP), поэтому в плагине есть маленький
**HTTP-over-pipe шим**: сам форматирует запрос и парсит ответ (`content-length` /
`chunked`). На каждый запрос открывается новый пайп с `Connection: close` — демон
отвечает и закрывает соединение, мы собираем ответ и парсим разом.

Опрос раз в N секунд:
- `GET /containers/json?all=1` — список контейнеров и их состояние;
- `GET /containers/{id}/stats?stream=false` — CPU/RAM по запущенным (если нужны мини-бары).

## Карточки
- **status «Docker»** — иконка + `N / M контейнеров`, бейдж = число запущенных;
  «демон недоступен», если Docker не запущен.
- **meters «контейнеры»** — топ-4 контейнера по CPU % или RAM % (настройка).

## Настройки
| Ключ | Тип | Назначение |
|------|-----|-----------|
| `endpoint` | string | TCP-адрес демона (напр. `127.0.0.1:2375`). **Пусто = именованный пайп** (рекомендуется). Через TCP — только если включён незащищённый эндпоинт демона |
| `card` | select | `both` / `status` / `meters` |
| `metric` | select | что в мини-барах: `cpu` или `mem` |
| `label` | string | подпись статус-карточки |
| `intervalSec` | number | период опроса, 2–300 с (по умолчанию 5) |
| `width`, `minW`, `maxW` | — | ширина карточек |

## Права
```json
"permissions": { "pipe": ["docker_engine"], "net": ["setting:endpoint"] }
```
`pipe` — доступ к пайпу движка (по умолчанию). `net:["setting:endpoint"]` — TCP-режим
разрешает только тот хост, что указан в настройке `endpoint`.

## Требования
- **Windows + Docker Desktop** (пайп `docker_engine` поднимает он сам, включать
  ничего не надо). Либо включённый TCP-эндпоинт демона и заполненная настройка
  `endpoint`.
- Unix-сокет `/var/run/docker.sock` не поддерживается (песочница без ФС; агент —
  Windows).

## Локальная проверка
Скопировать `manifest.json` + `plugin.js` в
`%AppData%\PcMonitorAgent\plugins\docker\1.0.1\`, включить плагин. При запущенном
Docker Desktop карточки появятся сразу; иначе статус покажет «демон недоступен».
