# MQTT-мост (Home Assistant)

Экспортирует выбранные данные Usee в Home Assistant через MQTT Discovery:
одно устройство «Usee», сущности — из дерева «Источники» в настройках плагина.
Мост **генерик**: он не знает ни одной карточки по имени — сущности строятся
из каталога шины значений (kind + ha-метаданные владельца поля). Установили
новый плагин с задекларированными `fields` — его источник сам появился в
дереве, удалили — его сущности штатно исчезли из HA.

## Настройки

- **Адрес брокера / порт / TLS / логин / пароль** — ваш MQTT-брокер
  (Mosquitto и т.п.). TLS проверяет сертификат по системным корням.
- **Имя устройства в HA** — как устройство называется на Device-странице.
- **Мин. интервал публикации** — троттлинг числовых/строковых значений на
  сущность (последнее значение серии доходит всегда); bool и уведомления —
  немедленно.
- **Источники** — дерево «карточка → поля»: галка на карточке = все её поля,
  включая будущие; раскрытие — тонкий выбор по полям.
- **Очистить сущности при выключении** — при выключении плагина отправить
  пустые retained-конфиги (сущности исчезнут из HA). Best-effort: успевает,
  пока соединение живо.

## Топики

- Конфиги: `homeassistant/<component>/usee_<id>/<object_id>/config` (retained)
- Доступность: `usee/<id>/availability` (`online` / LWT `offline`)
- Состояния: `usee/<id>/<object_id>/state`; протухший источник помечает свою
  сущность недоступной через `usee/<id>/<object_id>/availability`
- Уведомления Usee — событийная сущность (event) с типами
  notice/done/critical/achievement — годится в триггер автоматизации.
- На рестарт HA (birth `homeassistant/status` = online) мост переиздаёт
  discovery сам.

## Удаление самого моста — честный раздел

Удаление плагина рвёт соединение мгновенно — разослать пустые конфиги мост
уже не успевает, и сущности останутся в HA «недоступными» (retained-конфиги
живут в брокере). Правильный порядок:

1. Включите «Очистить сущности при выключении» и **выключите** плагин
   (чекбокс в списке) — сущности исчезнут из HA.
2. Потом удаляйте плагин.

Если удалили сразу — почистите вручную: удалите устройство «Usee» на странице
MQTT-интеграции в HA (HA сам сотрёт retained-конфиги), либо
`mosquitto_pub -r -n -t 'homeassistant/+/usee_<id>/+/config'` по каждому топику.
